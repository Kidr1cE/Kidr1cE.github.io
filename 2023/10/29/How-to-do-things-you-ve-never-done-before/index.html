Background

最近做了很多从来没接触过的工程问题，我发现使用一些方法能够更快的上手。在看了搜索技术 后，觉得应该写一个流程来配适工程问题，所以写了本篇文章

Function

离线部署K8s集群

已知的信息：

  1. 需求：在没有任何WLAN的情况下为部署K8s集群
  2. 可以使用一个叫Ansible的工具
  3. 可以使用一个叫Kubespray的工具

精简步骤

  1. Github搜索相关仓库，入门工具
  2. 使用AI+Google补齐相关术语及原理，理解你要做这件事的意义，了解需求
  3. Google搜索相关解决方法
  4. 遇到问题Issue -> Google -> AI -> 任何身边的大佬

根据场景的例子

  1. 首先查看了Kubespray仓库，Kubespray是使用Ansible工具为其他主机安装集群的一个工具，我发现其中提供了一个Offline环境的方法。开始使用相关方法进行测试
  2. 遇到问题：给只安装了系统的主机安装时存在Python版本、系统库缺失，无法完全断网
    • 查看Issue，有人提出问题，目前没有解决方法
    • 询问AI：Ansible安装过程中使用本机及目标机网络，鉴定为不满足需求
  3. Google搜索：k8s offline installation，发现kubespray-offline工具
    • 根据文档信息测试
    • 无法满足要求，无法自定义K8s版本
  4. 检查该项目，测试中发现满足完全断网需求
    • 项目由shell脚本实现
    • 结合项目及Kubespray可知，项目可以指定版本，是需要修改shell脚本即可
  5. 修改项目，使项目能够直接指定K8s版本
    • 全局检索未找到相关关键词
    • 根据项目结构定位相关文件
    • 结合AI及大佬亲情解读，Kubespray相关入参位置
  6. 测试，结合issue
    • 呃项目文档写错了，部分配置文件多加了/导致报错
    • 发现使用过程中 apt、Python 相关源均被替换为localhostDocker镜像仓库源，提出警告
    • 编写文档
